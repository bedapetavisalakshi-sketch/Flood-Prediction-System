# Non Functional Requirements

Non-functional requirements define the quality attributes and performance expectations of the Flood Prediction System.

## 1. Accuracy

The system should provide reliable flood predictions with good model accuracy.

## 2. Performance

The application should provide prediction results quickly after receiving user inputs.

## 3. Usability

The interface should be simple, user-friendly, and easy to understand.

## 4. Reliability

The system should work consistently without frequent failures.

## 5. Scalability

The system should support increasing amounts of data and future improvements.

## 6. Security

User inputs and system data should be handled securely.

## 7. Maintainability

The code should be organized properly so that future modifications and updates can be performed easily.
